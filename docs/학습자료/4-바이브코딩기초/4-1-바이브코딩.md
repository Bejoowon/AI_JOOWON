# 바이브코딩이란?

코딩을 몰라도 AI에게 "이거 만들어줘"라고 말할 수 있는 시대가 왔습니다.

---

## 바이브코딩이 뭔가요?

> **바이브코딩(Vibe Coding)** = 분위기/느낌으로 코딩을 시키는 것

코드를 직접 쓰는 게 아니라, **자연어(일상 말)로 AI에게 원하는 걸 설명**하면 AI가 코드를 만들어주는 방식입니다.

2025년 Andrej Karpathy(테슬라 AI 총괄 출신)가 처음 쓴 용어로, "코드를 쓰는 게 아니라 분위기를 전달한다"는 뜻입니다.

```
이전: 프로그래밍 언어를 배워서 직접 코드를 작성
바이브코딩: "이런 느낌으로 만들어줘"라고 말하면 AI가 코드를 작성
```

---

## 이전 방식 vs 바이브코딩

| | 이전 방식 | 바이브코딩 |
|---|----------|-----------|
| **필요한 것** | 프로그래밍 언어 학습 (몇 달~몇 년) | 원하는 걸 말로 설명하는 능력 |
| **작업 방식** | 코드를 한 줄씩 직접 작성 | AI에게 자연어로 지시 |
| **오류 대응** | 에러 메시지를 읽고 직접 디버깅 | "이거 에러 나는데 고쳐줘" |
| **진입장벽** | 높음 (코딩 교육 필수) | 낮음 (말만 할 줄 알면 됨) |
| **결과물** | 정확하지만 느림 | 빠르지만 검수 필요 |
| **비유** | 직접 요리하기 | 셰프에게 "이런 맛으로 만들어주세요" |

::: tip 핵심
바이브코딩은 **"코딩 실력"이 아니라 "설명 능력"이 중요한 시대**를 의미합니다.
:::

## 비개발자에게 왜 중요한가?

### 이전에는 불가능했던 것들

| 하고 싶은 것 | 이전 | 바이브코딩 시대 |
|------------|------|---------------|
| 간단한 웹페이지 만들기 | 외주 맡기거나 코딩 배워야 함 | "이런 페이지 만들어줘" |
| 엑셀 데이터 자동 분석 | VBA 매크로 학습 필요 | "이 엑셀 분석해서 그래프 만들어줘" |
| 반복 업무 자동화 | 개발자에게 요청 → 대기 | "매주 이 작업 자동으로 해줘" |
| 업무 도구 커스터마이징 | 불가능 (있는 그대로 사용) | "이 부분을 이렇게 바꿔줘" |

::: info 달라진 점
코딩을 몰라도 **앱, 자동화, 웹페이지**를 만들 수 있는 시대입니다.
필요한 건 "무엇을 만들고 싶은지" 명확하게 아는 것뿐입니다.
:::

## Claude Code에서 바이브코딩하는 3가지 도구

오늘 워크샵에서 배우는 3가지가 바로 바이브코딩의 핵심 도구입니다.

### 1. 스킬 (Skills) — "이렇게 해줘"를 저장

```
한 번 만들면 → /스킬이름 한 줄로 반복 실행
```

매번 같은 설명을 반복할 필요 없이, SKILL.md에 한 번 적어두면 끝.
바이브코딩의 **자동화 엔진**입니다.

### 2. CLAUDE.md — "나는 이런 사람이야"를 기억

```
한 번 써두면 → 모든 대화에서 자동 반영
```

AI에게 내 상황, 선호, 규칙을 알려두는 파일.
바이브코딩의 **컨텍스트(맥락)**입니다.

### 3. MCP — "저것도 연결해줘"로 확장

```
외부 도구 연결 → AI가 캘린더, 노션, 슬랙까지 직접 조작
```

AI의 손발을 늘려주는 연결 포트.
바이브코딩의 **확장 케이블**입니다.
