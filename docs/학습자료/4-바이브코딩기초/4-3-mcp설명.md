## USB-C 비유로 이해하기

노트북에 USB-C 포트가 있으면 다양한 장치를 꽂아서 기능을 늘릴 수 있죠.
MCP도 똑같습니다. AI에 연결하면 AI가 할 수 있는 일이 늘어납니다.

| 🔌 USB-C에 꽂는 것 | 생기는 능력 | 🤖 MCP에 연결하는 것 | AI가 얻는 능력 |
|:-----------------:|:-----------:|:-------------------:|:--------------:|
| 충전기 | 충전 | Google Calendar | 일정 관리 |
| 모니터 | 화면 확장 | Notion | DB 읽기/쓰기 |
| USB 메모리 | 파일 전송 | 파일 시스템 | 내 파일 접근 |
| 키보드 | 입력 장치 | Slack | 메시지 전송 |

## MCP 없을 때 vs MCP 있을 때

실제로 어떻게 다른지 대화로 느껴보세요.

### 😔 MCP 없을 때

> 👤 나: "오늘 오후 3시에 팀 미팅 잡아줘."
>
> 🤖 AI: "죄송해요, 저는 캘린더에 접근할 수 없어요. 직접 추가해주세요."

> 👤 나: "지난주에 보낸 이메일 좀 찾아줘."
>
> 🤖 AI: "저는 이메일에 접근할 수 없어요. 제목이나 내용을 직접 복사해주시면 도와드릴게요."

---

### 😊 MCP 있을 때 (Google Calendar 연결 후)

> 👤 나: "오늘 오후 3시에 팀 미팅 잡아줘."
>
> 🤖 AI: "네! 오늘 오후 3시에 '팀 미팅' 일정을 추가했습니다. 참석자 초대도 할까요?"

> 👤 나: "내일 비어있는 시간대 알려줘."
>
> 🤖 AI: "내일 오전 10시~12시, 오후 2시~4시가 비어있습니다. 어떤 일정 잡아드릴까요?"

::: tip MCP가 연결되면 달라지는 것
AI가 단순히 텍스트를 생성하는 것을 넘어서, **실제로 외부 서비스를 읽고, 쓰고, 실행**할 수 있게 됩니다.
"도와주는 챗봇"에서 "진짜 업무를 처리하는 비서"로 업그레이드되는 거예요.
:::

## MCP 서버와 Tools — 앱과 버튼

MCP를 더 정확하게 이해해볼까요?

**MCP 서버** = 스마트폰의 **앱**
**Tool** = 앱 안의 **기능 버튼**

예를 들어, 스마트폰에 "카카오톡" 앱을 설치하면 "메시지 보내기", "사진 보내기", "통화하기" 등 여러 기능을 쓸 수 있죠?

MCP도 같습니다:

```
📱 Google Calendar MCP 서버 (= 앱)
├── 🔧 일정 조회 (Tool)
├── 🔧 일정 생성 (Tool)
├── 🔧 일정 수정 (Tool)
└── 🔧 일정 삭제 (Tool)

📱 Notion MCP 서버 (= 앱)
├── 🔧 페이지 읽기 (Tool)
├── 🔧 페이지 생성 (Tool)
├── 🔧 데이터베이스 조회 (Tool)
└── 🔧 댓글 추가 (Tool)
```

::: tip 핵심 구분
- **MCP 서버 1개를 연결** = 앱 1개 설치
- **Tool 여러 개 사용 가능** = 앱 안의 기능 여러 개 사용
:::

## 이런 말을 하면 → 이 Tool이 호출됩니다

MCP가 연결되어 있으면, 자연어로 말해도 AI가 알아서 맞는 Tool을 골라 실행합니다.

| 💬 내가 하는 말 | 🔧 호출되는 Tool | 📱 MCP 서버 |
|:--------------|:--------------|:-----------|
| "내일 일정 알려줘" | `list-events` | Google Calendar |
| "오후 3시에 미팅 잡아줘" | `create-event` | Google Calendar |
| "프로젝트 DB에서 진행중인 것만 보여줘" | `notion-search` | Notion |
| "이 내용으로 노션 페이지 만들어줘" | `notion-create-pages` | Notion |
| "드라이브에서 보고서 파일 찾아줘" | `search-files` | Google Drive |
| "#general 채널에 메시지 보내줘" | `send-message` | Slack |

::: info 마법이 아닙니다
AI가 "내일 일정"이라는 키워드를 보고, 연결된 MCP 서버 중 Calendar의 `list-events` Tool이 적합하다고 **자동 판단**하는 겁니다. 여러분은 그냥 자연어로 말하면 됩니다!
:::

## 관심 있으면 여기서 시작하세요

준비가 됐다 싶을 때, 이 순서대로 따라오세요.

```
① 스킬을 2~3주 실제 업무에서 써본다
      ↓
② "AI가 캘린더까지 써줬으면 좋겠다" 싶은 순간이 온다
      ↓
③ Google Calendar MCP부터 시작한다 (가장 쉽고 실용적)
      ↓
④ 설정 가이드 한 줄씩 따라한다 (터미널 명령어 1줄로 설치)
      ↓
⑤ "내일 오전 10시 팀 미팅 잡아줘" 가 실제로 된다 🎉
```

::: info 설정 방식은 어렵지 않아요
대부분의 MCP 설정은 이 3단계를 따릅니다.

1. MCP 서버 설치 (터미널 명령어 한 줄)
2. 설정 파일에 서비스 정보 입력
3. Claude Code 재시작

가이드만 따라하면 비개발자도 충분히 할 수 있는 수준입니다.
궁금한 점은 커뮤니티에서 편하게 질문해주세요!
:::
