# ⭐ CLAUDE.md란?

이전 페이지에서 잠깐 "AI의 기억"이라고 소개했죠? 여기서 자세히 알아봅시다.

---

## Claude의 "뇌" 비유

매번 대화할 때마다 "나는 마케팅팀이고, 한국어로, 존댓말로..." 설명하기 지치지 않나요?

CLAUDE.md는 **AI에게 심어주는 기억 칩**입니다. 한 번 설정하면 매번 설명할 필요 없이 AI가 기억합니다.

| 비유 | CLAUDE.md 없을 때 | CLAUDE.md 있을 때 |
|------|------------------|------------------|
| 신입 사원 | 매일 아침 "우리 팀은 이래요" 설명 | 첫날 OT만 하면 다음부턴 알아서 |
| 비서 | "한국어로, 존댓말로..." 매번 지시 | 한 번 알려주면 영구 반영 |
| 네비게이션 | 매번 목적지 입력 | 자주 가는 곳 즐겨찾기 저장 |

:::tip 핵심
CLAUDE.md = **"나는 이런 사람이고, 이렇게 일해줘"** 를 적는 파일
:::

---

## 언제 읽히나요?

Claude Code를 시작하면 **가장 먼저** CLAUDE.md를 읽습니다. 대화의 시작점이죠.

```
Claude Code 시작
      ↓
① CLAUDE.md 자동 로드 (있으면)
      ↓
② 사용자의 메시지를 받음
      ↓
③ CLAUDE.md 내용 + 사용자 메시지를 합쳐서 응답
```

:::info 매번 읽습니다
새 대화를 시작할 때마다 CLAUDE.md를 읽습니다. 그래서 한 번 써두면 **모든 대화에** 자동 적용됩니다.
:::

---

## 어떤 내용이 들어가나요?

정해진 형식은 없지만, 보통 이런 내용을 적습니다:

| 카테고리 | 예시 | 효과 |
|---------|------|------|
| **내 정보** | "나는 셀루션랩 마케팅 담당자" | AI가 맥락을 이해 |
| **언어/톤** | "한국어, 비즈니스 존댓말" | 매번 지시 불필요 |
| **문서 형식** | "보고서: 제목→요약→상세→액션아이템" | 일관된 결과물 |
| **금지 사항** | "영어 사용 금지, 이모지 금지" | 실수 방지 |
| **프로젝트 정보** | "Next.js 앱, TypeScript 사용" | 정확한 코드 생성 |
| **자주 쓰는 규칙** | "파일명은 kebab-case" | 반복 지시 불필요 |

## 실제 CLAUDE.md 예시

비개발자가 바로 쓸 수 있는 예시입니다:

```markdown
# CLAUDE.md

## 내 정보
- 이름: 최주원
- 소속: 셀루션랩 마케팅팀
- 역할: 콘텐츠 마케터

## 작성 규칙
- 모든 문서는 한국어로 작성
- 비즈니스 존댓말 사용
- 간결하게, 3문장 이내로 요약 먼저

## 보고서 형식
1. 제목
2. 한 줄 요약
3. 상세 내용
4. 액션아이템 (담당자 + 기한)

## 자주 하는 업무
- 블로그 포스트 초안 작성
- 주간 마케팅 리포트
- 거래처 이메일 커뮤니케이션
```

:::tip 이렇게 써두면
"이메일 써줘"라고만 해도 AI가 한국어 + 존댓말 + 간결한 스타일로 작성합니다. 매번 톤이나 형식을 지정할 필요가 없어요!
:::

## /init 명령어 — 자동으로 만들어주기

CLAUDE.md를 처음부터 직접 쓸 필요 없습니다. Claude Code에게 시키면 됩니다!

### 사용 방법

Claude Code 채팅창에 입력:
```
/init
```

### 자동 분석 항목

/init을 실행하면 Claude가 현재 프로젝트를 분석해서 자동으로 CLAUDE.md를 만들어줍니다:

```
/init 실행
    ↓
① 프로젝트 폴더 구조 분석
    ↓
② 사용 중인 기술/도구 파악
    ↓
③ 기존 설정 파일 확인
    ↓
④ CLAUDE.md 자동 생성!
```

| 분석 항목 | 감지하는 것 | CLAUDE.md에 반영 |
|----------|-----------|----------------|
| 폴더 구조 | src/, docs/, tests/ 등 | 프로젝트 구조 설명 |
| 패키지 파일 | package.json, requirements.txt | 기술 스택, 빌드 명령어 |
| 설정 파일 | .eslintrc, tsconfig.json | 코딩 규칙 |
| README | README.md | 프로젝트 목적 |

:::warning 비개발자라면
/init은 주로 코드 프로젝트에서 유용합니다. 문서 작업 위주라면 **위의 예시처럼 직접 작성**하는 게 더 정확합니다!
:::

## 저장 위치 — 어디에 넣나요?

CLAUDE.md는 **두 가지 위치**에 저장할 수 있습니다:

```
🏠 개인 설정 (어디서든 적용)
   ~/.claude/CLAUDE.md
   → "한국어로 작성, 존댓말 사용" 같은 개인 습관

📁 프로젝트 설정 (이 폴더에서만 적용)
   내프로젝트/CLAUDE.md
   → "보고서 형식은 이거, 팀원 이름은..." 같은 프로젝트 규칙
```

### 둘 다 있으면?

합쳐집니다! 프로젝트 설정이 개인 설정보다 우선합니다.

```
개인 설정: "한국어, 존댓말"
     +
프로젝트 설정: "보고서 형식은 A, 팀원은 B"
     =
AI가 보는 것: "한국어, 존댓말, 보고서 형식은 A, 팀원은 B"
```

:::tip 추천 전략
1. **개인 CLAUDE.md** (`~/.claude/CLAUDE.md`): 내 스타일, 언어, 톤
2. **프로젝트 CLAUDE.md** (`프로젝트/CLAUDE.md`): 프로젝트별 규칙, 형식
:::

## 잘 쓰는 4가지 습관

### 습관 1: 요청으로 수정하기

CLAUDE.md를 직접 편집할 필요 없습니다. Claude에게 시키세요!

```
"CLAUDE.md에 '이메일은 항상 5문장 이내' 규칙을 추가해줘"
```

### 습관 2: 실수를 지침으로 바꾸기

AI가 실수했을 때가 CLAUDE.md를 보강할 기회입니다.

```
AI가 영어로 답변함
    ↓
"앞으로 절대 영어로 답변하지 마" 라고 지시
    ↓
"이 규칙을 CLAUDE.md에도 추가해줘"
    ↓
다음부터 영구 적용!
```

### 습관 3: 파일 위치 알려주기

AI가 파일을 못 찾으면 위치를 알려주세요.

```markdown
## 프로젝트 구조
- 보고서 템플릿: /templates/report.md
- 고객 데이터: /data/customers.csv
- 주간보고 저장: /reports/weekly/
```

### 습관 4: 상황별로 업데이트하기

CLAUDE.md는 살아있는 문서입니다. 업무가 바뀌면 같이 바꾸세요.

```
"CLAUDE.md에서 '마케팅팀'을 '전략기획팀'으로 바꿔줘"
```

## 핵심 정리

| 질문 | 답변 |
|------|------|
| CLAUDE.md가 뭔가요? | AI에게 심어주는 기억 파일 |
| 꼭 만들어야 하나요? | 아니요, 없어도 됨. 있으면 훨씬 편함 |
| 어떻게 만드나요? | 직접 작성 또는 `/init` 명령어 |
| 어디에 저장하나요? | 프로젝트 루트 또는 `~/.claude/` |
| 언제 읽히나요? | Claude Code 시작할 때 자동으로 |
| 수정은 어떻게? | Claude에게 "CLAUDE.md 수정해줘"라고 시키면 됨 |

:::tip 기억하세요
CLAUDE.md는 **한 번 쓰면 매번 효과**가 나는 투자입니다.

5분 투자해서 CLAUDE.md를 만들면, 앞으로 매 대화에서 30초씩 절약됩니다.

스킬이 "이 업무를 자동화"라면, CLAUDE.md는 "나를 이해시키기"입니다.
:::
