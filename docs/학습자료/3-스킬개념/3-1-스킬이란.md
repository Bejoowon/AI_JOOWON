# 스킬이란?

오늘 세미나의 핵심은 **스킬**입니다. 이것만 제대로 이해하면 나머지는 자연스럽게 따라옵니다.

---

## 스킬이 뭔가요?

Claude Code 공식 사이트의 정의를 빌리면 이렇습니다:

> "Skills는 Claude가 할 수 있는 것을 확장합니다. SKILL.md 파일을 작성하고 지침을 포함하면 Claude가 이를 자신의 도구 모음에 추가합니다."

말이 좀 어렵죠? 쉽게 한 줄로 정의하면

**스킬 = AI에게 주는 업무지침서(SOP)**


 신입사원이 왔다고 생각을 해보겠습니다.
 우리가 AI 채팅창을 열면, 딱 신입이 막 입사한 것과 동일한 것입니다.


![](/attachments/Pasted-image-20260303001309.png)
**인수인계도 없이 다짜고짜 말하면**
```
나: "이 엑셀에 현재 상황 업데이트 해주시고 메일로 보내주세요."
신입: "어떤 형식으로요? 어디서 업데이트 하죠? 누구한테 보내요?"
나: (매번 처음부터 설명...)
```


**인수인계도하고 SOP도 전달하면**
```
나: "이 엑셀에 현재 상황 업데이트 해주시고 메일로 보내주세요."
신입: "네,Workspace에서 현재 상황 받은다음 업데이트하여 수신처로 메일 보내겠습니다."
나: (편안-)
```



프롬프트를 입력해서 요청을 할 때 스킬 있고 없고 차이는 아래 처럼 선명해요.

| 스킬 없이 😰        | 스킬 있으면 ✨              |
| --------------- | --------------------- |
| 매번 긴 프롬프트 작성    | **한 마디**로 실행          |
| "지난번에 어떻게 했더라?" | **항상 같은 방식**으로 처리     |
| 결과물 들쭉날쭉        | **일관된** 품질            |
| 나만 아는 방법        | **팀과 공유** 가능          |
| 설명하다 지침         | Claude가 **알아서** 단계 수행 |

::: tip 가장 중요한 포인트
스킬은 **코딩이 아닙니다**. 글로 적는 겁니다. "이런 상황에서, 이런 형식으로, 이런 규칙을 지켜서 결과를 만들어줘"라고 글로 적어두면 AI가 그대로 실행합니다.
:::

## 스킬의 정체: SKILL.md 파일

스킬의 실체는 폴더이구요! 
그 안의 필수는 **SKILL.md라는 텍스트 파일 하나**입니다. 
더 간단히 생각하면, **업무할 수 있는 메모**라고 보시면 됩니다

![](/attachments/Pasted-image-20260303002346.png)



### 실제 SKILL.md 맛보기

```yaml
---
name: meeting-summary
description: |
  긴 회의록을 분석하여 핵심 결정사항과 액션아이템을 구조화된 마크다운으로 추출하는 스킬.
  회의록 텍스트를 직접 붙여넣거나 파일 경로를 지정하면 요약을 생성한다.

  Triggers:
  - "회의록 요약", "회의록 정리", "미팅노트 요약", "meeting summary"
  - 사용자가 회의록 텍스트를 붙여넣고 요약/정리를 요청할 때
  - 사용자가 회의록 파일 경로를 주고 요약/정리를 요청할 때
---

# 회의록 요약

## 입력 처리

- **텍스트 직접 입력**: 사용자가 회의록을 대화창에 붙여넣은 경우 그대로 분석
- **파일 경로**: 사용자가 파일 경로를 제공한 경우 Read 도구로 읽은 후 분석

## 출력 형식

아래 마크다운 형식으로 출력한다.

```markdown
# 회의 요약

- **회의명**: (파악 가능한 경우)
- **일시**: (파악 가능한 경우)
- **참석자**: (파악 가능한 경우)

## 핵심 결정사항

1. [결정사항]
2. ...

## 액션아이템

| 담당자 | 기한 | 내용 |
|--------|------|------|
| 홍길동 | 3/15 | 디자인 시안 제출 |
| 김철수 | 3/20 | API 연동 개발 완료 |

## 주요 논의사항

- [논의 내용]
- ...
```


### SKILL.md의 핵심 구조

| 부분       | 역할                    | 예시                |
| -------- | --------------------- | ----------------- |
| `---` 사이 | **메타정보** (언제 이 스킬 쓸지) | name, description |
| `---` 아래 | **본문** (어떻게 처리할지)     | 단계, 주의사항 등        |

**가장 중요한 것**: `description`에 **언제 쓰는지** 적어두면 Claude가 **알아서 인식**!

```yaml
description: 회의록/미팅노트/인터뷰 기록을... "회의록 요약해줘"라고 말하면 실행.
```

이렇게 적어두면 사용자가 "회의록 정리해줘"라고만 해도 이 스킬이 자동으로 활성화됩니다.


### Skill을 발동하는 2가지 방법
1. `description`에 적어둔대로 트리거 문구를 활용한다
   ex) "회의록 요약해줘". "이미지 만들어줘" 등
2. 스킬 커맨드를 직접 입력한다. /meeting-summary

## 💡 점진적 공개: 스킬이 똑똑한 이유

스킬은 **필요할 때만** 정보를 불러옵니다. 매번 모든 지침을 다 읽지 않아요.

### 3단계 로딩

```
1단계: 메타정보(name, description)만 읽음 (항상)
       → "이 스킬은 피드백 정리할 때 쓰는구나"

2단계: 본문 읽음 (관련 있을 때만)
       → 사용자가 "피드백 정리해줘" 하면 그때 읽음

3단계: 참고 파일 (필요할 때만)
       → references/ 폴더 내용은 스킬을 진짜 실행할 때만
```

### 왜 이게 중요한가요?

|방식|토큰 사용|속도|
|---|---|---|
|매번 긴 프롬프트|매번 전체 사용 📈|느림|
|스킬 (점진적 공개)|필요한 만큼만 📉|빠름|

**비유**: 백과사전 전체를 매번 읽는 것 vs 목차 보고 필요한 페이지만 펼치는 것

## 📁 스킬이 저장되는 위치

스킬은 **두 군데**에 저장할 수 있습니다.

> 📚 **공식 문서**: [code.claude.com/docs/en/skills](https://code.claude.com/docs/en/skills)

### 전역 설치 vs 프로젝트 설치

| 구분                 | 위치                      | 적용 범위                  |
| ------------------ | ----------------------- | ---------------------- |
| **전역 (Personal)**  | `~/.claude/skills/스킬명/` | 내 컴퓨터의 모든 프로젝트(git공유X) |
| **프로젝트 (Project)** | `.claude/skills/스킬명/`   | 이 프로젝트 폴더에서만(git에 공유)  |
| **프로젝트 (Local)**   | `.claude/skills/스킬명/`   | 이 프로젝트 폴더에서만(+나만!)     |
![](/attachments/Pasted-image-20260303002919.png)


전역은 내 컴퓨터의 사용자(User)가 사용할 수 있도록 설정하는 것이고
프로젝트는 내 컴퓨터의 지정폴더 내 에서만 사용할 수 있도록 설정하는 것입니다.

AI를 집으로 초대했다 -> 전역 설정
AI를 집에 있는 방으로 초대했다 -> 프로젝트


### 전역 설치, 프로젝트 설치 어떻게 구분해야 할까?

#### 예시 /billing 스킬
영수증을 자동으로 처리하는 스킬을 만들었다고 가정합니다.
이 스킬은 영수증 관련 업무만 진행할 떄 작동되면 좋을 것이에요.

그래서 이런 경우엔 전역 설치보단, 해당 업무를 진행하는 폴더에서만 설치합니다.

| 설치 위치       | 결과                                              |
| ----------- | ----------------------------------------------- |
| **전역 설치**   | 다른 프로젝트에서 작업 중 "청구서 정리해줘"라고 말하면 갑자기 실행될 수 있음 😰 |
| **프로젝트 설치** | 영수증 프로젝트 폴더에서만 작동. 다른 곳에서는 트리거 안 됨 ✅            |


### 스킬 작동 우선순위

전역 설치, 프로젝트 설치 헷갈린대 만약에 같은 이름의 스킬이 두 곳에 있으면?

```
Enterprise > 전역(Personal) > 프로젝트(Project)
```

전역이 우선 적용됩니다. (공식 문서 기준)
