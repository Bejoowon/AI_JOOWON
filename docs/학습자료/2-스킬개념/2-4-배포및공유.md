# 배포 및 공유

만든 스킬을 나만 쓰기 아깝지 않나요? GitHub에 올려서 누구나 쓸 수 있게 공유해봅시다.

---

## 왜 배포하나요?

| | 로컬에만 두기 | GitHub에 올리기 |
|---|--------------|----------------|
| **백업** | 내 컴퓨터 고장 나면 사라짐 | 클라우드에 영구 보관 |
| **공유** | USB로 복사해야 함 | 링크 하나로 공유 |
| **협업** | 나만 사용 가능 | 팀 전체가 설치 가능 |
| **버전 관리** | 수정하면 이전 버전 사라짐 | 모든 변경 이력 보존 |
| **포트폴리오** | 나만 알고 있음 | "나는 AI 자동화를 할 줄 아는 사람" 증명 |

::: tip 핵심
스킬을 GitHub에 올리면 **백업 + 공유 + 포트폴리오**가 한 번에 해결됩니다.
:::

---

## 실습 1: GitHub CLI 설치 & 로그인

GitHub에 파일을 올리려면 먼저 준비가 필요합니다. 걱정 마세요 — **Claude Code에게 자연어로 시키면 됩니다.**

### Step 1: GitHub 계정 만들기 (없는 경우)

1. [github.com](https://github.com) 접속
2. **Sign up** 클릭
3. 이메일, 비밀번호, 사용자명 입력
4. 무료 플랜으로 충분합니다

::: tip 이미 계정이 있다면
바로 Step 2로 넘어가세요!
:::

### Step 2: GitHub CLI 설치

Claude Code 채팅창에 입력하세요:

```
GitHub CLI(gh)를 설치해줘
```

Claude Code가 알아서 운영체제에 맞는 설치 명령어를 실행합니다.

### Step 3: GitHub 로그인

```
GitHub에 로그인해줘
```

브라우저가 열리면서 인증 화면이 나옵니다. **Authorize** 버튼을 클릭하면 완료!

::: tip 체크포인트
Claude Code에게 "GitHub 로그인 상태 확인해줘"라고 물어보세요.
"Logged in as 사용자명" 이 나오면 성공입니다.
:::

## 실습 2: 내 스킬 GitHub에 올리기

이제 아까 만든 이메일초안 스킬을 GitHub에 올려봅시다.

### Step 1: 저장소 만들기

Claude Code에게 이렇게 말하세요:

```
내 스킬을 GitHub에 올리고 싶어.
my-claude-skills 라는 이름으로 GitHub 저장소를 만들고,
.claude/skills/ 폴더의 내용을 올려줘
```

### Step 2: 확인하기

Claude Code가 작업을 마치면 GitHub 저장소 URL을 알려줍니다.

```
https://github.com/내사용자명/my-claude-skills
```

브라우저에서 접속해보세요. 내 스킬 파일들이 올라가 있는 걸 확인할 수 있습니다!

::: info 이게 전부입니다
Git 명령어를 직접 칠 필요 없습니다. Claude Code에게 자연어로 말하면 `git init`, `git add`, `git commit`, `git push`를 전부 대신 해줍니다.
:::

## 실습 3: 친구 스킬 설치하기

이번엔 반대로, **다른 사람의 스킬을 내 컴퓨터에 설치**해봅시다.

### Step 1: 상대방의 저장소 URL 받기

옆 사람에게 GitHub 저장소 URL을 받으세요.

```
https://github.com/친구사용자명/my-claude-skills
```

### Step 2: 설치하기

Claude Code에게:

```
이 GitHub 저장소에서 스킬을 설치해줘:
https://github.com/친구사용자명/my-claude-skills
```

### Step 3: 테스트

설치된 스킬을 실행해보세요. 다른 사람이 만든 스킬이 내 컴퓨터에서 동작합니다!

::: tip 스킬 공유의 힘
"이메일초안" 스킬 하나를 만들어서 공유하면:
- 팀원 5명이 각자 쓸 수 있고
- 누가 개선하면 다 같이 업데이트할 수 있고
- 새로 입사한 사람도 바로 팀의 스킬을 쓸 수 있습니다
:::

## 스킬 공유의 의미 — 팀 활용 시나리오

혼자 쓰는 것도 좋지만, 팀에서 쓰면 효과가 배가 됩니다.

### 시나리오: 마케팅팀 5명이 같은 스킬을 쓴다면?

```
팀장이 "고객응대" 스킬을 만듦
      ↓
GitHub에 올림
      ↓
팀원 4명이 각자 설치
      ↓
모든 팀원이 동일한 톤, 동일한 품질로 고객 응대
      ↓
한 명이 스킬을 개선하면 → 전체 업데이트
```

| 항목 | 스킬 공유 전 | 스킬 공유 후 |
|------|------------|------------|
| 이메일 톤 | 사람마다 제각각 | 팀 전체 일관성 |
| 신입 온보딩 | "이렇게 써, 저렇게 써" 교육 | "이 스킬 설치해" 한마디 |
| 품질 관리 | 매번 검수 필요 | 스킬이 품질 보장 |

## CLI 어려운 경우: 대안

터미널이 정 어렵다면 **웹에서 직접 업로드**하는 방법도 있습니다.

### GitHub 웹으로 업로드하기

1. [github.com](https://github.com) 로그인
2. 오른쪽 상단 **+** → **New repository** 클릭
3. Repository name: `my-claude-skills` 입력
4. **Create repository** 클릭
5. **uploading an existing file** 링크 클릭
6. SKILL.md 파일을 드래그 앤 드롭
7. **Commit changes** 클릭

::: info 결과는 같습니다
CLI든 웹이든, GitHub 저장소에 SKILL.md 파일이 올라가는 것은 동일합니다.
편한 방법을 선택하세요!
:::
