# 스킬이란?

오늘 워크샵의 핵심은 **스킬**입니다. 이것만 제대로 이해하면 나머지는 자연스럽게 따라옵니다.

---

## 스킬이 뭔가요?

Claude Code 공식 사이트의 정의를 빌리면 이렇습니다:

> "Skills는 Claude가 할 수 있는 것을 확장합니다. SKILL.md 파일을 작성하고 지침을 포함하면 Claude가 이를 자신의 도구 모음에 추가합니다."

말이 좀 어렵죠? 쉽게 바꾸면:

> **"AI에게 주는 업무 매뉴얼. 한 번 만들면 `/스킬이름` 한 줄로 계속 재사용."**

가장 중요한 포인트는 이겁니다. 스킬은 **코딩이 아닙니다**. 글로 적는 겁니다. "이런 상황에서, 이런 형식으로, 이런 규칙을 지켜서 결과를 만들어줘"라고 글로 적어두면 AI가 그대로 실행합니다.

![스킬은 AI에게 주는 업무 매뉴얼](/images/skill-manual.png)

---

## Before / After로 체감하기

말보다 비교가 빠릅니다. 실제 업무 상황 3가지입니다.

| 상황 | 스킬 없이 (Before) | 스킬 있으면 (After) |
|------|-------------------|-------------------|
| **이메일 쓸 때** | "존댓말로, 요점 먼저, 3-5문장으로..." 매번 설명 | `/이메일초안 납품 일정 변경 요청` 한 줄 끝 |
| **회의록 정리** | 회의 끝나고 30분 동안 수동 정리 | `/회의록요약` → 3분 만에 정리 완료 |
| **주간보고** | 매주 금요일마다 형식 고민부터 시작 | `/주간보고 이번 주 한 일: ...` → 바로 완성 |
| **결과 품질** | 그날 컨디션에 따라 들쭉날쭉 | 항상 같은 형식, 같은 퀄리티 |
| **소요 시간** | 매번 10~30분 | 매번 1~3분 |

---

## 스킬의 정체: SKILL.md 파일

스킬의 실체는 **SKILL.md라는 텍스트 파일 하나**입니다. 이 파일은 두 부분으로 구성됩니다.

### 파일 구조

1. **YAML frontmatter** (`---` 사이): 스킬의 이름표. 이름과 설명을 적습니다.
2. **마크다운 본문**: 실제 지침. 입력 형식, 출력 형식, 규칙을 적습니다.

### 실제 SKILL.md 예시

비개발자도 바로 쓸 수 있는 이메일 초안 스킬입니다.

```yaml
---
name: 이메일초안
description: |
  비즈니스 이메일 초안을 작성하는 스킬.
  "/이메일초안 [상황]" 또는 "이메일 써줘"로 호출.
---

# 이메일 초안 작성기

## 입력
사용자가 이메일을 보내야 하는 상황을 설명합니다.

## 출력 형식
📧 이메일 초안

받는 사람: [상황에서 추론]
제목: [간결한 제목]
[본문 3-5문장]

## 작성 규칙
1. 비즈니스 존댓말 사용
2. 간결하게 (3-5문장)
3. 핵심 정보 빠짐없이 포함
```

### SKILL.md 구조 해부

| 부분 | 역할 | 비유 | 필수? |
|------|------|------|-------|
| `---` 사이 (frontmatter) | 이름표 | 명함 | 권장 |
| `name` | 스킬 이름, `/이름`으로 호출 | 별명 | 권장 |
| `description` | AI가 언제 쓸지 판단하는 설명 | 자기소개 | 권장 |
| 마크다운 본문 | 실제 지침 | 업무 매뉴얼 본문 | 필수 |

---

## 스킬이 저장되는 위치

| 위치 | 경로 | 적용 범위 |
|------|------|-----------|
| 개인 (글로벌) | `~/.claude/skills/스킬이름/SKILL.md` | 어디서든 사용 가능 |
| 프로젝트 | `.claude/skills/스킬이름/SKILL.md` | 이 프로젝트에서만 |

:::info 어디에 만들어야 하나요?
처음에는 **프로젝트 폴더** 안에 만드세요 (`.claude/skills/`).
자주 쓰는 스킬은 나중에 글로벌(`~/.claude/skills/`)로 옮기면 됩니다.
:::

### 설치 위치 시나리오: "billing 스킬을 어디에 만들어야 할까?"

예를 들어 회사 청구서를 만드는 `billing` 스킬이 있다고 합시다.

- **A 프로젝트에서만 쓰는 청구서 양식** → `.claude/skills/billing/SKILL.md` (프로젝트)
- **어떤 프로젝트에서든 쓰는 개인 스킬** → `~/.claude/skills/billing/SKILL.md` (글로벌)

:::warning 잘못된 위치에 설치하면?
- **글로벌에 프로젝트 전용 스킬을 설치** → 다른 프로젝트에서 엉뚱한 청구서 양식이 적용되어 혼동
- **프로젝트에만 설치** → 다른 프로젝트에서는 이 스킬을 쓸 수 없음

처음에는 프로젝트에 만들고, 여러 곳에서 필요해지면 글로벌로 옮기세요!
:::

### 우선순위 규칙

같은 이름의 스킬이 여러 곳에 있으면 어떻게 될까요? Claude Code는 **위에서 아래로** 우선순위를 적용합니다.

```
Enterprise (회사 관리자) — 최우선
   └── Organization (팀) — 두번째
      └── 글로벌 (~/.claude/) — 세번째
         └── 프로젝트 (.claude/) — 네번째
```

:::info 비개발자를 위한 요약
대부분의 경우 Enterprise나 Organization은 신경 쓸 필요 없습니다. 우리는 **글로벌**과 **프로젝트** 두 가지만 기억하면 됩니다!
:::

## 스킬은 어떻게 읽히나요?

스킬이 한 번에 전부 로딩되는 건 아닙니다. Claude Code는 **3단계에 걸쳐 점진적으로** 스킬을 읽습니다.

```
1단계: description만 읽음 (가볍게 훑기)
      ↓ 관련 있다고 판단하면
2단계: SKILL.md 전문 읽음 (지침 파악)
      ↓ 실행 시작
3단계: 지침대로 작업 수행
```

왜 이렇게 나눠서 읽을까요? 스킬이 10개, 20개로 늘어나면 매번 전부 읽기엔 비효율적이기 때문입니다. 그래서 **description(설명)만 먼저 훑어보고**, 지금 요청에 관련된 스킬만 골라서 전문을 읽습니다.

| 방식 | 매번 드는 비용 | 장점 |
|------|-------------|------|
| 매번 프롬프트 입력 | 매번 전체 지침 토큰 소비 | 유연함 |
| 스킬 사용 | description만 항상 로드 + 필요할 때만 전문 읽음 | 효율적, 일관성 |

:::tip 그래서 description이 중요한 겁니다
description은 AI가 "이 스킬이 지금 필요한가?"를 판단하는 **유일한 단서**입니다. 여기에 언제 쓰는 스킬인지, 어떤 키워드로 호출하는지를 명확히 적어야 AI가 정확히 골라냅니다.
:::

## 스킬이 실행되는 흐름

```
1. 사용자가 "/이메일초안 납품 일정 변경 요청" 입력
          ↓
2. Claude Code가 .claude/skills/이메일초안/SKILL.md 찾기
          ↓
3. SKILL.md 내용을 읽고 지침 따르기
          ↓
4. 지침대로 이메일 초안 생성!
```

:::tip 핵심 포인트
스킬은 두 가지 방법으로 실행됩니다.

1. **직접 호출**: `/이메일초안 상황설명` — 내가 직접 부르는 것
2. **자동 감지**: "이메일 좀 써줘" — AI가 description을 보고 관련 스킬을 자동 적용
:::

## 좋은 스킬 vs 나쁜 스킬

| | 좋은 스킬 | 나쁜 스킬 |
|--|-----------|----------|
| 범위 | 딱 한 가지 일만 함 | "뭐든 다 해줘" |
| 출력 형식 | 명확한 형식 지정 | "알아서 해" |
| 규칙 | 구체적 (3-5문장, 존댓말) | 모호한 ("잘 써줘") |
| 이름 | 짧고 직관적 (이메일초안, 주간보고) | 길고 복잡 (만능AI자동화도구v2) |

:::warning 가장 흔한 실수
"너무 많은 걸 하려고" 하는 것입니다.
처음 스킬은 딱 한 가지 일만 하게 만드세요. 잘 되면 나중에 기능을 추가하면 됩니다.
:::

## 스킬 말고도 있나요? — Claude 확장 기능 비교

스킬이 Claude Code를 확장하는 유일한 방법은 아닙니다. 전체 그림을 한 번 보고 가겠습니다.

| 기능 | 무엇인가요? | 비유 | 오늘 다루나요? |
|------|-----------|------|-------------|
| **스킬 (Skills)** | AI 업무 매뉴얼 (SKILL.md) | 요리 레시피 | :star: 핵심! |
| **CLAUDE.md** | AI의 기억/성격 설정 | 직원 명찰 | :star: 간단히 |
| **MCP** | 외부 도구 연결 포트 | USB-C 케이블 | 소개만 |
| **서브에이전트** | AI가 부르는 보조 AI | 팀 리더가 부르는 팀원 | - |
| **커스텀 슬래시 명령** | 빠른 동작 단축키 | 키보드 단축키 | - |
| **훅 (Hooks)** | 특정 이벤트에 자동 실행 | 알람 설정 | - |

:::tip 오늘은 스킬에 집중!
나머지 기능들은 스킬을 마스터한 뒤 자연스럽게 알게 됩니다. 스킬이 Claude Code 확장의 출발점이에요.
:::
