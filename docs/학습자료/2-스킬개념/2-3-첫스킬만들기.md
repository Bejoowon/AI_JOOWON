# 첫 스킬 만들기

## 실습 시간!

오늘 실습의 핵심입니다. 함께 **실제로 동작하는 스킬**을 하나 만들어봅시다.

처음이라 긴장되시나요? 괜찮습니다. 한 단계씩 같이 합니다. 끝까지 따라오시면 **오늘 바로 써먹을 수 있는 나만의 도구**가 생깁니다.

## 오늘 만들 것

![스킬 만들기 4단계: 폴더 생성 → 작성 → 테스트 → 성공](/images/skill-creation-flow.png)

### 이메일 초안 작성기

> **상황만 말하면** 비즈니스 이메일 초안을 만들어주는 스킬

**사용 예시**
```
/이메일초안 거래처에 납품 일정 2주 연기 요청
```

**결과**
```
📧 이메일 초안

받는 사람: ABC 담당자님
제목: 납품 일정 조정 요청의 건

안녕하세요. 항상 좋은 협력 감사드립니다.

다름이 아니라 이번 납품 일정과 관련하여 말씀드릴 사항이 있어
연락드립니다. 내부 사정으로 인해 기존 일정보다 2주가량 납품이
늦어질 것으로 예상됩니다.

불편을 드려 대단히 죄송하며, 조정된 일정으로 협의가 가능하신지
확인 부탁드리겠습니다.

감사합니다.
[이름] 드림
```

이메일 초안 작성, 매번 머릿속에서 끌어내기 번거롭지 않으셨나요? 이제 스킬이 해줍니다.

## Step 1: 스킬 폴더 만들기

⏱️ 예상 시간: 3분 | 난이도: 쉬움

스킬은 특정 폴더 구조 안에 파일로 저장됩니다.

```
내프로젝트/
└── .claude/
    └── skills/
        └── 이메일초안/
            └── SKILL.md    ← 이 파일이 스킬의 본체
```

::: tip 직접 만들 필요 없어요!
Claude Code에게 시키면 됩니다.
:::

::: info 지금 해보세요
Claude Code 채팅창에 아래 내용을 **그대로 복사해서** 입력해보세요.
:::

```
이메일초안이라는 스킬 폴더를 만들어줘.
.claude/skills/이메일초안/SKILL.md 파일을 생성해
```

::: tip 체크포인트 — 이게 보이면 성공!
"파일을 생성했습니다" 또는 "폴더를 만들었습니다"는 메시지가 나왔나요? 그럼 완료입니다!
:::

::: details 폴더가 이미 있다고 하면?
그대로 진행하시면 됩니다. 이미 만들어진 것이니 Step 2로 넘어가세요.
:::

## Step 2: SKILL.md 작성하기

⏱️ 예상 시간: 10분 | 난이도: 보통

이제 스킬의 본체를 작성합니다. 함께 한 줄씩 살펴볼게요.

::: info 지금 해보세요
아래 내용 전체를 **복사해서** Claude Code 채팅창에 붙여넣으세요.
:::

```
.claude/skills/이메일초안/SKILL.md 파일에 아래 내용을 넣어줘:

---
name: 이메일초안
description: |
  비즈니스 이메일 초안을 작성하는 스킬.
  "/이메일초안 [상황]" 또는 "이메일 써줘"로 호출.
---

# 이메일 초안 작성기

## 입력
사용자가 이메일을 보내야 하는 **상황**을 설명합니다.
$ARGUMENTS에 상황이 주어지면 그 내용을 기반으로 초안을 작성합니다.

## 출력 형식
아래 형식으로 이메일 초안을 작성합니다:

### 📧 이메일 초안

**받는 사람**: [상황에서 추론]
**제목**: [간결한 제목]

---

[본문]

- 첫 문장: 인사 + 용건
- 중간: 구체적 내용 (날짜, 숫자 등 포함)
- 마지막: 요청사항 + 마무리 인사

---

## 작성 규칙
1. 비즈니스 존댓말 사용
2. 간결하게 (3-5문장)
3. 핵심 정보 빠짐없이 포함
4. 과도한 미사여구 금지
```

### SKILL.md 구조 살펴보기

파일을 저장했으면 잠깐 구조를 같이 봅시다. 이 구조가 **스킬의 설계도**입니다.

#### frontmatter (--- 사이 부분) — 스킬의 이름표

```yaml
---
name: 이메일초안
description: |
  비즈니스 이메일 초안을 작성하는 스킬.
  "/이메일초안 [상황]" 또는 "이메일 써줘"로 호출.
---
```

각 필드가 하는 일:

| 필드 | 역할 | 쉽게 말하면 |
|------|------|------------|
| `name` | 스킬 이름. `/이름`으로 호출할 때 쓰는 이름 | 스킬의 호출 코드 |
| `description` | AI가 이 스킬을 **언제 자동으로 쓸지** 판단하는 설명 | 스킬의 자기소개서 |

::: tip description이 핵심입니다
"이메일 써줘"라고 자연어로 말해도 AI가 description을 읽고 "아, 이메일초안 스킬을 써야겠구나!" 하고 자동으로 연결합니다. description을 잘 쓸수록 자동 매칭이 정확해집니다.
:::

#### 본문 — 스킬의 동작 지침

| 부분 | 역할 | 비유 |
|--------|--------|--------|
| `## 입력` | AI에게 "이런 걸 받아" | 주문서 |
| `## 출력 형식` | AI에게 "이렇게 보여줘" | 양식지 |
| `## 작성 규칙` | AI에게 "이 규칙 지켜" | 지침서 |

::: tip 핵심 포인트
이 세 가지 구조(입력 / 출력 형식 / 작성 규칙)만 기억하면 **어떤 스킬이든 만들 수 있습니다!**
:::

### 스킬을 부르는 두 가지 방법

**방법 1: 직접 호출 (슬래시 명령어)**
```
/이메일초안 거래처에 납품 일정 변경 요청
```
→ 확실하게 이 스킬을 사용

**방법 2: 자연어로 말하기**
```
이메일 좀 써줘, 거래처에 납품 일정 변경 요청
```
→ AI가 description을 보고 자동으로 스킬 매칭!

### 슬래시 뒤에 쓰는 말 = $ARGUMENTS

```
/이메일초안 거래처에 납품 일정 변경 요청
            ↑ 이 부분이 $ARGUMENTS
```

SKILL.md 본문에서 `$ARGUMENTS`를 쓰면, 슬래시 뒤에 입력한 내용이 자동으로 그 자리에 들어갑니다. 마치 빈칸 채우기처럼요.

::: tip 체크포인트 — 이게 보이면 성공!
"SKILL.md에 내용을 저장했습니다"는 응답이 왔나요? 파일이 생성된 거 맞습니다. Step 3으로 가세요!
:::

## Step 3: 테스트하기

⏱️ 예상 시간: 5분 | 난이도: 쉬움 (재미있는 단계!)

만든 스킬을 실제로 써봅시다. 두근두근하죠? 이제 AI가 내 규칙대로 움직이는 걸 눈으로 확인하는 순간입니다. 자신감 갖고 해보세요!

### 테스트 1 — 기본 사용

::: info 지금 해보세요
:::

```
/이메일초안 거래처 ABC에 납품 일정을 2주 연기해달라고 요청
```

결과가 나왔나요? 형식이 맞게 나왔는지 확인해보세요.

::: tip 체크포인트 — 이게 보이면 성공!
"📧 이메일 초안" 형식으로 결과가 나왔나요? 훌륭합니다!
:::

### 테스트 2 — 다른 상황으로

::: info 지금 해보세요
:::

```
/이메일초안 팀장님에게 내일 오전 반차 사용 요청
```

같은 스킬인데 전혀 다른 상황에도 잘 동작하나요? 이게 바로 스킬의 힘입니다.

### 테스트 3 — 자유 테스트

::: info 지금 해보세요 — 이 단계가 제일 중요합니다!
여러분의 실제 업무에서 보내야 할 이메일 상황으로 직접 테스트해보세요.
"잘 동작한다"는 건 **내 상황에서** 잘 동작한다는 뜻입니다.
:::

## Step 4: 수정하기

⏱️ 예상 시간: 5분 | 난이도: 보통 (핵심 원리를 배우는 단계!)

결과가 마음에 완전히 들지 않아도 정상입니다. 처음부터 완벽한 스킬은 없습니다. **스킬은 쓰면서 다듬는 것**, 그게 올바른 방식입니다. 지금부터 그 방법을 배워봅시다.

### 수정 예시

| 이런 문제가 있다면 | SKILL.md 이렇게 수정하세요 |
|---------------------|------------------------------|
| "이메일이 너무 딱딱해요" | 작성 규칙에 "적절히 부드러운 톤 사용" 추가 |
| "영어 이메일도 됐으면 좋겠어요" | 작성 규칙에 "사용자가 영어를 요청하면 영어로 작성" 추가 |
| "본문이 너무 짧아요" | 작성 규칙의 "3-5문장"을 "5-7문장"으로 변경 |

::: info 지금 해보세요
Claude Code에게 수정을 시켜봅시다.
:::

```
이메일초안 스킬을 수정해줘.
작성 규칙에 "적절히 부드러운 톤 사용"을 추가해줘
```

수정 후 다시 테스트해보세요. 달라졌나요?

::: tip 체크포인트 — 이게 보이면 성공!
수정 전과 후 결과 톤이 달라졌나요? 그럼 여러분은 이제 AI를 원하는 대로 조율할 수 있는 겁니다!
:::

::: info 핵심 원리
**SKILL.md를 수정하면 AI의 행동이 바뀝니다.**

이것이 스킬의 힘입니다 — 코딩 없이, 글만으로 AI를 내 입맛대로 커스터마이징!
:::
