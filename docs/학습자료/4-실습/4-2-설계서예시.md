> 앞서 배운 **간소화 스킬 설계 템플릿**을 실제로 채운 완성 예시입니다.
> "아, 이렇게 쓰면 되는구나!" 하고 참고만 하세요. 정답은 없습니다.

## 완성된 설계서: SNS 캡션 생성기

::: info 이런 분이 만들었어요
인스타그램을 운영하는 소상공인 A씨. 새 제품을 올릴 때마다 캡션 문구를 30분씩 고민하다가 지쳐서 스킬을 만들기로 했습니다.
:::

```markdown
# 나의 스킬 설계서: SNS 캡션 생성기

## 1. 선언
- **스킬 이름**: SNS캡션
- **한 줄 설명**: 제품/서비스 정보를 주면 인스타그램·쓰레드용 캡션을 만들어주는 스킬
- **MVP 목표**: "제품 정보를 입력하면 → SNS 캡션 초안을 만들어준다"

## 2. 언제 쓰나요?
- **대표 상황**: 새 제품 출시할 때 인스타 포스팅 문구를 고민하는 시간이 너무 오래 걸림
- **왜 필요한가**: 매번 30분씩 캡션 고민 → 5분으로 줄이고 싶다

## 3. 사용법
- **이렇게 부르면**: "/SNS캡션 새 세럼 출시, 비타민C 함유, 29,000원"
- **결과물**: 인스타그램용 캡션 + 해시태그 (복사해서 바로 쓸 수 있는 형태)

## 4. 범위
- **하는 것**:
  1. 인스타그램/쓰레드용 캡션 생성
  2. 관련 해시태그 5-10개 추천
  3. 이모지 적절히 포함
- **안 하는 것**:
  1. 이미지 생성이나 편집
  2. 직접 SNS에 게시

## 5. 완료 기준
"이렇게 되면 성공": 캡션을 복사해서 인스타에 바로 붙여넣기 가능
```

::: tip 설계서 작성 꿀팁
"안 하는 것"을 명확히 쓰는 게 핵심입니다. AI는 범위를 좁혀줄수록 더 정확한 결과를 냅니다.
"뭐든지 다 해줘" 보다 "이것만 해줘"가 훨씬 좋습니다.
:::

## Before → After: 설계서가 SKILL.md로 바뀌면?

설계서는 **사람이 읽는 기획서**, SKILL.md는 **AI가 읽는 명령서**입니다.
설계서를 잘 써두면 Claude Code에게 한 마디면 끝납니다.

```
"이 설계서 보고 SKILL.md 만들어줘"
```

::: details 완성된 SKILL.md 보기 (클릭해서 펼쳐보세요)

```markdown
---
name: SNS캡션
description: |
  제품/서비스 정보로 인스타그램·쓰레드용 캡션을 생성하는 스킬.
  "/SNS캡션 [제품 정보]" 또는 "캡션 만들어줘"로 호출.
---

# SNS 캡션 생성기

## 입력
사용자가 **제품/서비스 정보**를 제공합니다.
- 제품명, 특징, 가격 등

## 출력 형식

### 📱 인스타그램 캡션

[훅이 되는 첫 줄]

[제품 소개 2-3문장]

[CTA (행동 유도)]

---
[해시태그 5-10개]

## 작성 규칙
1. 첫 줄은 호기심을 유발하는 문장
2. 이모지 적절히 사용 (과하지 않게)
3. 해시태그는 인기 태그 + 니치 태그 조합
4. 200자 내외 (인스타 미리보기 기준)
5. 캐주얼한 말투 사용
```

:::

::: info 설계서 vs SKILL.md — 한 눈에 비교

| 구분 | 설계서 | SKILL.md |
|------|--------|----------|
| 읽는 대상 | 나 (사람) | Claude Code (AI) |
| 언제 씀 | 기획·설계할 때 | 실제 실행할 때 |
| 형식 | 자유롭게 | 정해진 양식 |
| 오늘 한 것 | ✅ 직접 작성 | Claude Code에게 맡기면 됩니다 |

설계서만 잘 써두면, SKILL.md 변환은 Claude Code가 대신 해줍니다!
:::

## 스킬 배포 & 공유

만든 스킬을 다른 사람과 나누는 방법입니다.

### 방법 1: 프로젝트에 포함 (가장 쉬움)

`.claude/skills/` 폴더를 프로젝트와 함께 공유하면 됩니다.
GitHub, 구글 드라이브, USB... 어떤 방식이든 OK.

```
내프로젝트/
└── .claude/
    └── skills/
        └── 이메일초안/
            └── SKILL.md    ← 이 파일만 있으면 됨
```

받는 사람이 이 프로젝트 폴더를 열면 자동으로 스킬이 인식됩니다.

### 방법 2: 글로벌 스킬로 설치

자주 쓰는 스킬은 개인 폴더에 넣으면 **어디서든** 사용 가능:

```
~/.claude/skills/이메일초안/SKILL.md
```

이렇게 하면 어떤 프로젝트를 열든 `/이메일초안` 사용 가능!

### 방법 3: 팀에 배포

팀원 전체가 같은 스킬을 쓰려면:
1. `.claude/skills/` 폴더를 Git 저장소에 커밋
2. 팀원이 프로젝트를 clone하면 자동으로 스킬 포함

:::tip 오늘은 여기까지!
배포는 스킬을 잘 만든 다음의 이야기입니다.
오늘은 "내가 쓸 스킬 하나 만들기"에 집중하세요.
나중에 잘 만들어진 스킬을 팀에 공유하면 **팀 전체의 업무가 자동화**됩니다!
:::
